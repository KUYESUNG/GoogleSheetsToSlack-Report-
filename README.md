# 🧾 Slack 자동 테스트 진행률 보고 스크립트

## 📌 개요

이 스크립트는 **테스트 진행 현황을 자동으로 Slack에 보고**하는 Google
Apps Script입니다.\
`Summary` 시트에 기록된 테스트 결과를 요약해 Slack으로 전송함으로써,\
테스트 매니저와 이해관계자들이 **별도의 문서 접근 없이** 프로젝트의 진행
상태를 한눈에 파악할 수 있도록 합니다.

------------------------------------------------------------------------

## 🎯 제작 배경

QA 업무에서 테스트 케이스 수는 프로젝트 규모에 따라 급격히 늘어납니다.\
진행률은 `Summary` 시트에서 확인할 수 있지만,\
테스트 일정이 겹치거나 병렬 진행되는 경우 **모든 시트를 직접 열어
확인해야 하는 비효율**이 발생합니다.

이 스크립트는 이러한 번거로움을 줄이기 위해\
**테스트 진행률과 성공률을 자동으로 Slack에 보고**하도록
설계되었습니다.\
이를 통해 매니저는 **시트를 일일이 확인하지 않고도**\
프로젝트의 품질 상태를 실시간으로 확인할 수 있습니다.

------------------------------------------------------------------------

## ⚙️ 작동 방식

1.  **설정 관리**
    -   `PropertiesService`를 이용해 `SPREADSHEET_ID`와
        `SLACK_WEBHOOK_URL`을 한 번만 등록해두면,\
        이후 코드에서 자동으로 참조합니다.
2.  **데이터 수집**
    -   `Summary` 시트의 B18행부터 데이터를 읽어 `"Total"` 행 전까지의
        각 항목을 수집합니다.\
    -   각 시트별 Pass, Fail, Block, 진행률 등의 정보를 배열 형태로
        정리합니다.
3.  **진행률 계산**
    -   `"Test Case Performance Rate"`와 `"Test Success Rate"` 항목의
        값을 추출해\
        전체 진행률과 성공률을 계산합니다.
4.  **Slack 메시지 구성 및 전송**
    -   수집한 데이터를 표 형태로 정렬한 후 코드 블록(\`\`\`) 형태로
        포맷팅하여\
        Slack Webhook URL로 전송합니다.\
    -   메시지는 한눈에 보기 쉬운 구조로 표시되어 즉시 진행 상황을
        파악할 수 있습니다.

------------------------------------------------------------------------

## 🚀 기대 효과
- **테스트 현황을 실시간으로 공유**할 수 있어 커뮤니케이션 효율이 향상됩니다.  
- **매니저의 관리 부담을 감소**시켜, 테스트 일정과 품질 지표를 빠르게 파악할 수 있습니다.  
- **QA 엔지니어는 데이터 집계보다 분석에 집중**할 수 있게 되어 업무 생산성이 높아집니다.  
- 테스트 결과를 **하나의 메시지로 요약해 전달**함으로써, 대규모 프로젝트에서도 명확한 품질 인사이트를 제공합니다.
